The pattern — always the same loop at every stage:
comment → label → regenerate → repeat → advance
The terminal action is always a human merge. Only the repo owner can trigger the pipeline.
Commands

Four skills, one workflow

Install the plugin once, use these commands from any directory in Claude Code.

🏗️ /polis:set-up Scaffold a brand-new Polis-powered repo from anywhere. Walks you through every step interactively.
1Repo — name, directory, owner, visibility
2Secrets — PAT, Claude OAuth, optional backend keys
3Config — review or customise polis.yml
4Labels — pipeline trigger labels created automatically
🎮 /polis:drive A terminal cockpit for the whole pipeline — no GitHub web UI needed.
→State panel — issues & PRs with current stage
→Create issues and apply agent:* labels
→Watch Actions runs, read AI review feedback
→Merge finished PRs — confirms before every action
📝 /polis:docs Sync in-repo documentation with code changes since the last git tag. Run after any meaningful change or when cutting a release.
1README — add, update, or remove changed sections
2CHANGELOG — draft a new entry from the commit log
3Inline docs — flag stale or missing docstrings
🌐 /polis:page Publish official documentation to a dedicated public GitHub repo and deploy it to GitHub Pages — keeping the main project private.
1Visualization — update the pipeline diagram page
2Docs sync — copy README, CHANGELOG, agent personas
3Pages — commit, push, and trigger the deploy workflow

Label reference

Add these labels to an issue or PR to trigger the agent pipeline.

LabelWhat the agent doesOutput
agent:archWrites a high-level architecture doc with a phased work breakdownDraft PR
agent:rearchRewrites the architecture doc from your PR comments (repeat as needed)Updated draft PR
agent:decomposeCreates one milestone per phase and one issue per breakdown lineGitHub Issues
agent:specWrites a detailed spec including a Test planDraft PR
agent:respecRewrites the spec from your PR comments (repeat as needed)Updated draft PR
agent:codeImplements code + tests, runs an AI review loop, marks the PR readyPR ready
agent:fixRevises from PR comments — or fixes failing tests if no comment — then re-runs testsUpdated PR
agent:code_reviewAI review of the code PR with the default review recipeReview comments
agent:spec_review:<backend>Reviews the spec using a named backend (e.g. codex)Review comments
agent:arch_review:<profile>Reviews the arch doc with a named profile (e.g. thorough)Review comments
